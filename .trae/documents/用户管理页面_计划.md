# 用户管理页面（用户设置）实现计划

## Summary
- 在现有 Flask + SQLAlchemy 管理后台中新增“用户管理”菜单入口，并提供“用户设置”页面用于新增/编辑/删除可登录账号。
- 为 `users` 表补充“姓名”字段（可选），用于标记账号使用人。
- 暂不引入权限/角色体系：所有已登录用户均可访问并管理用户（按当前决定）。

## Current State Analysis
- 登录体系：`flask-login`，登录/登出路由位于 [auth.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/routes/auth.py#L1-L73)，`login_manager` 在 [app.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/app.py#L19-L51) 初始化。
- 用户模型：`User(id, username, password_hash, created_at)` 位于 [models.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/models.py#L15-L24)；启动时通过 [bootstrap.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/services/bootstrap.py#L27-L55) 确保管理员账号存在。
- 顶部菜单：集中渲染在 [base.html](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/templates/base.html#L142-L159)，当前只有“商品资料”下拉与“退出”。
- 数据库建表方式：`Base.metadata.create_all()`；该方式不会为既有表自动补列，因此新增字段需要额外 schema 兜底（参考项目里 `ensure_drug_update_unique_index()` 的做法）。

## Proposed Changes

### 1) 数据模型：为 users 增加“姓名”字段
- 文件： [models.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/models.py)
- 变更：
  - 在 `User` 模型新增 `name`（或等价命名）字段：`String(64)`，可为空（与“姓名可选”的决定一致）。
  - 其他现有逻辑（登录校验、user_loader）无需调整。

### 2) 启动时 schema 兜底：为既有数据库补充 users.name 列
- 文件： [bootstrap.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/services/bootstrap.py)
- 变更：
  - 新增 `ensure_user_columns()`（或同等函数名），启动时执行：
    - `ALTER TABLE users ADD COLUMN IF NOT EXISTS name VARCHAR(64);`
  - 在 `create_app()` 启动流程中调用该函数（紧随 `create_all()` 之后、蓝图注册之前均可）。
- 目的：
  - 避免生产/已有库因缺列导致页面查询或更新失败。

### 3) 新增“用户管理 > 用户设置”菜单入口
- 文件： [base.html](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/templates/base.html#L142-L159)
- 变更：
  - 在现有“商品资料”旁新增一个 dropdown：
    - 顶级菜单名：`用户管理`
    - 子菜单：`用户设置`
    - 链接指向新增蓝图的 endpoint（例如 `url_for('user_mgmt.user_settings')`）

### 4) 新增用户管理蓝图与路由（CRUD）
- 新文件：`webapp/routes/user_mgmt.py`
- 注册： [app.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/app.py#L43-L49)
- 路由设计（均 `@login_required`）：
  - `GET /users/settings`
    - 展示用户列表 + 新增表单
    - 可选支持 `?edit_id=<id>` 进入编辑态（或用独立 edit 页；实现时二选一并固定）
  - `POST /users/create`
    - 新增用户：`username`（必填、唯一）、`password`（必填）、`name`（可选）
  - `POST /users/update/<id>`
    - 编辑用户：允许修改 `username`、`name`；`password` 允许“留空则不修改”
  - `POST /users/delete/<id>`
    - 删除用户
    - 保护规则：禁止删除当前登录用户（避免自删导致会话异常/锁死），返回页面错误提示
- 校验与错误处理：
  - 用户名去空格、禁止空字符串
  - 用户名唯一冲突：友好错误提示
  - 密码确认：创建与“填写了新密码的更新”都需要 `password_confirm` 一致
  - 密码散列：使用 `werkzeug.security.generate_password_hash`（项目已使用）

### 5) 新增模板：用户设置页面
- 新文件：`webapp/templates/user_settings.html`
- UI 结构（沿用现有 Semantic UI 风格，参考 [medical_info.html](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/templates/medical_info.html#L1-L120)）：
  - 页面标题与说明
  - 消息区：成功/失败提示（negative/positive message）
  - 用户列表表格：
    - 列：ID、用户名、姓名、创建时间、操作
    - 操作：编辑（进入编辑态/编辑页）、删除（POST）
  - 新增用户表单：
    - 用户名、姓名（可选）、密码、确认密码、提交按钮
  - 编辑用户表单（当进入编辑态时展示）：
    - 用户名、姓名、（可选）新密码 + 确认新密码、保存按钮

## Assumptions & Decisions
- 操作范围：新增 + 编辑 + 删除（已确认）。
- 姓名字段：可选（已确认）。
- 访问控制：所有登录用户均可访问“用户设置”（已确认），暂不增加权限系统。
- 不修改 `api/medical_info.py`（项目硬约束，且本需求无关）。

## Verification
- 启动：
  - 使用 uv 运行：`uv run webapp/app.py`
- 手工验收用例：
  - 登录后顶部菜单出现“用户管理 > 用户设置”，点击可进入页面。
  - 新增用户：用户名唯一校验生效；密码确认不一致会报错；成功后可用新账号登录。
  - 编辑用户：可修改用户名/姓名；不填新密码则不变；填新密码需二次确认。
  - 删除用户：可删除非当前用户；尝试删除当前用户会被阻止并提示原因。
