# 用户登录日志页面（Plan）

## Summary
- 在“用户管理”下新增“用户登录日志”页面：支持按**日期范围**与**用户名**筛选，并分页展示登录时间记录。
- 新增一张独立数据库表 `user_login_logs` 存储日志（仅记录**成功登录**），包含 `username`、`login_at`、`ip` 等字段。
- 登录成功时自动写入一条登录日志。

- 用户管理位于 [user_mgmt.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/routes/user_mgmt.py) 与 [user_settings.html](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/templates/user_settings.html)：目前仅“用户设置”。
- 数据库初始化位于 [app.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/app.py)：`Base.metadata.create_all()` 在启动时建表；对少量结构变更/索引使用 `ensure_*` 函数补齐（见 [bootstrap.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/services/bootstrap.py)）。

- 现有分页与每页条数约束在 [config.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/config.py)（默认 100，可选 100/200/500），商品查询页已实现分页交互（可复用体验）。

## Assumptions & Decisions
- 仅记录**成功登录**（用户确认）。
- 日志表额外记录 **IP**（用户确认，取 `request.remote_addr`）。
- 页面需要分页，默认 100 条/页，可选值与商品查询页一致（用户确认）。
- 日期范围筛选使用“日期”（`YYYY-MM-DD`），实现为：`login_at >= start_date 00:00:00` 且 `login_at < (end_date + 1 day) 00:00:00`（包含 end_date 当日）。
- 用户名筛选：使用“包含匹配”（`ILIKE %keyword%`），便于模糊查询；如需改为精确匹配，可在实现阶段调整为 `==`。

## Proposed Changes

### 1) 新增登录日志表模型
**File**: [models.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/models.py)
- 新增 `UserLoginLog` 模型（表名 `user_login_logs`）：
  - `id`：自增主键
  - `user_id`：可空整数（记录用户 ID，便于关联；不强制外键，避免删除用户带来约束问题）
  - `username`：登录用户名（冗余保存，避免用户删除后无法展示）
  - `ip`：可空字符串（建议 64 长度）
  - `login_at`：登录时间（`DateTime(timezone=True)`，`server_default=func.now()`）

### 2) 确保索引存在（可选但推荐）
**Files**:
- [bootstrap.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/services/bootstrap.py)
- [app.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/app.py)

- 在 `bootstrap.py` 新增 `ensure_user_login_log_indexes()`：
  - `CREATE INDEX IF NOT EXISTS ix_user_login_logs_login_at ON user_login_logs (login_at DESC)`
  - `CREATE INDEX IF NOT EXISTS ix_user_login_logs_username ON user_login_logs (username)`
- 在 `create_app()` 中 `Base.metadata.create_all()` 后调用该函数，保证旧库升级时索引能补齐。

### 3) 登录成功时写入日志
**File**: [auth.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/routes/auth.py)
- 在用户名/密码校验通过后、`login_user(user)` 前后（同一个 `session_scope` 内）新增 `UserLoginLog` 插入：
  - `user_id=user.id`
  - `username=user.username`
  - `ip=request.remote_addr`
  - `login_at` 由数据库默认值填充（或显式写 `datetime.now(timezone.utc)`，实现阶段二选一）
- 注意不记录密码、不记录明文敏感信息。

### 4) 新增“用户登录日志”页面路由与查询逻辑
**File**: [user_mgmt.py](file:///Users/lucifer/code/Python/TRAE/personal-toolkit/webapp/routes/user_mgmt.py)
- 新增路由：`GET /users/login-logs`（`@login_required`）
- Query 参数：
  - `start_date`（可空，`YYYY-MM-DD`）
  - `end_date`（可空，`YYYY-MM-DD`）
  - `username`（可空，模糊匹配）
  - `page`（默认 1）
  - `page_size`（默认 `Settings.default_page_size`；若不在 `allowed_page_sizes` 内则回落默认）
- 查询实现要点：
  - 构建 SQLAlchemy Query：按 `login_at DESC` 排序
  - 统计总数 `total`
  - 按 `LIMIT/OFFSET` 分页取当页数据
  - `total_pages = ceil(total / page_size)` 并用于页面显示分页组件
- 错误处理：
  - 日期解析失败：页面提示“日期格式错误”，并忽略该条件（或直接回落为空）。
